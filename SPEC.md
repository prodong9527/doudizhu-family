# 斗地主AI对战小程序 - 规格文档

## 1. 项目概述

### 项目名称

**斗地主 · 段位之战**

### 项目类型

- 单页HTML应用（前后端不分离）
- 静态托管于GitHub Pages
- 纯前端 + 云端AI API

### 核心功能

一款带有排位赛系统的斗地主AI对战游戏，玩家与AI进行对战，通过赢得对局获取星星和积分，提升段位。

### 目标用户

- 家庭成员娱乐使用
- 轻度游戏玩家

## 2. 技术方案

### 架构选择

**可行方案分析：**

| 方案               | 数据存储        | 成本     | 复杂度 | 适合度        |
| ---------------- | ----------- | ------ | --- | ---------- |
| 纯前端+LocalStorage | 浏览器本地       | 免费     | 低   | 中（无法多设备同步） |
| 前端+Firebase      | Firebase云端  | 免费套餐足够 | 中   | 极高（推荐）     |
| 前端+JSONBin       | JSONBin.io  | 免费     | 低   | 高          |
| 静态托管+Backendless | Backendless | 免费套餐   | 中   | 高          |

**最终推荐：Firebase + LocalStorage混合方案**

- LocalStorage：本地快速读写，临时数据
- Firebase Firestore：云端用户数据持久化（支持多设备同步）
- 优点：免费、功能强大、接入简单、支持匿名登录

### 技术栈

- **前端框架**：原生HTML5 + CSS3 + JavaScript（ES6+）
- **AI集成**：MiniMax2.7 API（通过后端代理或直接调用）
- **数据存储**：Firebase Firestore（云端）+ LocalStorage（本地缓存）
- **音频**：Web Audio API（点击音效、背景音乐）
- **动画**：CSS3 Animation + Transition
- **部署**：GitHub Pages（静态托管）

### API集成说明

- MiniMax API需要后端代理（因为API密钥不能暴露在前端）
- 可选方案：
  1. 使用Firebase Cloud Functions作为代理
  2. 使用Cloudflare Workers免费代理
  3. 直接使用Firebase Realtime Database规则限制

## 3. 游戏规则

### 斗地主基本规则

- **玩家人数**：3人（1个玩家 + 2个AI）
- **发牌**：每人17张，留3张底牌
- **叫地主**：玩家可选择叫/不叫地主
- **出牌**：单张、对子、三张、顺子、连对、飞机、炸弹、火箭等

### 排位赛规则

#### 段位系统（从低到高）

| 段位 | 档位数量 | 每档星星数 | 难度描述     |
| -- | ---- | ----- | -------- |
| 青铜 | 3档   | 3星/档  | AI会犯明显错误 |
| 白银 | 3档   | 4星/档  | AI偶尔犯错   |
| 黄金 | 5档   | 5星/档  | AI基本无误   |
| 铂金 | 5档   | 5星/档  | AI较强     |
| 翡翠 | 6档   | 6星/档  | AI很强     |
| 钻石 | 6档   | 6星/档  | AI很强     |
| 星耀 | 6档   | 6星/档  | AI非常强    |

#### 星星获取规则

| 胜利情况         | 获得星星 | 说明   |
| ------------ | ---- | ---- |
| 完胜（AI剩余>10张） | 3星   | 大获全胜 |
| 险胜（AI剩余≤10张） | 2星   | 艰难取胜 |
| 惜败（剩余1张）     | 1星   | 差一点  |
| 其他失败         | 0星   | -    |

#### 星星扣除规则

- 失败时：扣除1星（段位最低档不扣星）

#### AI难度设计

| 段位 | AI行为特征         |
| -- | -------------- |
| 青铜 | 随机出牌、不记牌、可能拆炸弹 |
| 白银 | 基本按规则出牌、偶尔失误   |
| 黄金 | 正常对战水平、会算牌     |
| 铂金 | 较强、正常对战水平、会记牌  |
| 翡翠 | 较强、会记牌、善于配合    |
| 钻石 | 仅次于顶级水平、较优出牌策略 |
| 星耀 | 顶级水平、最优出牌策略    |

## 4. 积分系统

### 积分获取

| 行为   | 积分变化     |
| ---- | -------- |
| 每局胜利 | +50      |
| 每局失败 | +10（安慰奖） |
| 完胜额外 | +30      |
| 每日首胜 | +20      |

### 积分用途

- 皮肤商店：消耗积分购买牌背皮肤
- 未来扩展：更多功能

## 5. UI/UX设计

### 整体风格

- **主题**：浅色系、艺术文艺风格
- **配色**：米白背景 + 深棕文字 + 淡金色点缀
- **字体**：思源宋体/楷体（中文）+ 无衬线英文
- **图标**：简约线条风格，拒绝emoji

### 页面结构

#### 1. 登录/注册页

- 简洁输入框（用户名/密码）
- 登录/注册按钮（带动画反馈）
- 艺术装饰元素（淡雅花纹边框）
- 也有游客模式，但数据存储是一次性的，下次登录就没了

#### 2. 主页面

- 顶部：用户信息栏（头像、用户名、段位、星星）
- 中部：段位展示卡（当前位置可视化）
- 底部：功能按钮（开始匹配、商店、我的）

#### 3. 游戏房间

- 顶部：对手信息（头像、血量）
- 中部：扑克牌区域（玩家手牌）
- 底部：出牌按钮、提示按钮
- 左侧/右侧：AI玩家手牌（背面显示）
- 中间：出牌区域（当前出的牌）

#### 4. 商店页

- 牌背皮肤展示（网格布局）
- 积分余额显示
- 购买/使用按钮

### 动画设计

| 交互    | 动画效果                 |
| ----- | -------------------- |
| 按钮点击  | 缩放+阴影变化（0.95倍，100ms） |
| 卡牌选中  | 上移+发光边框（150ms）       |
| 卡牌出牌  | 飞入出牌区（300ms）         |
| 胜利/失败 | 全屏特效（星星/沮丧动画）        |
| 段位晋升  | 光辉特效+弹窗              |

### 音效设计

- **点击音效**：轻柔点击声（0.1s）
- **出牌音效**：卡片放置声（0.2s）
- **胜利音效**：欢快铃声（1s）
- **失败音效**：低沉音效（1s）
- **背景音乐**：可选轻音乐（可关闭）

## 6. 数据模型

### 用户数据结构（Firebase）

```javascript
{
  users: {
    [uid]: {
      username: string,
      passwordHash: string, // 加密存储
      rank: {
        level: number, // 0-青铜, 1-白银, 2-黄金, 3-铂金, 4-钻石
        tier: number, // 0-3（每段位的档位）
        stars: number, // 当前档位星星 0-3
        totalStars: number // 累计获得星星
      },
      points: number, // 积分
      skins: string[], // 已拥有皮肤ID列表
      currentSkin: string, // 当前使用皮肤
      stats: {
        totalGames: number,
        wins: number,
        perfectWins: number // 完胜次数
      },
      createdAt: timestamp,
      lastLoginAt: timestamp
    }
  }
}
```

### 皮肤数据结构

```javascript
{
  skins: {
    [skinId]: {
      id: string,
      name: string,
      type: "cardBack", // 牌背
      cost: number,
      preview: string // 预览图URL
    }
  }
}
```

## 8. 扑克牌资源

### 资源路径

- 原皮PNG：`原皮高清全套扑克牌/PNG/`
- 原皮GIF：`原皮高清全套扑克牌/GIF/`

### 卡牌命名规则

- 黑桃：Spade + 点数（2-A, 10, J, Q, K）
- 红心：Heart + 点数
- 梅花：Club + 点数
- 方块：Diamond + 点数
- 大王：JOKER-A
- 小王：JOKER-B
- 背景/牌背：Background

### 素材使用计划

- **游戏用牌**：PNG格式（清晰、高质量）
- **商店预览**：GIF格式（动态效果）
- **初始皮肤**：仅原皮（Background.png作为默认牌背）

## 9. 安全考虑

### API密钥保护

- MiniMax API Token不得暴露在前端
- 使用Firebase Cloud Functions或Cloudflare Workers代理
- 设置API调用频率限制

### 数据安全

- 密码加密存储（bcrypt）
- Firebase安全规则限制读写权限
- 用户只能修改自己的数据

***

## 10. 部署方案

### GitHub Pages部署

1. 创建GitHub仓库
2. 将HTML/CSS/JS文件推送到仓库
3. 在Settings中启用GitHub Pages
4. 访问：`https://[username].github.io/[repo-name]/`

### Firebase Hosting（备选）

- 更快的加载速度
- 自动HTTPS
- 免费套餐足够家庭使用
